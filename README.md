# Yandex Cloud Balance для Home Assistant

Кастомная интеграция Home Assistant для отображения баланса биллингового аккаунта Yandex Cloud.

Интеграция получает IAM-токен по OAuth-токену Яндекса, запрашивает список биллинговых аккаунтов через Yandex Cloud Billing API и создаёт сенсор баланса в Home Assistant.

## Возможности

- Настройка через интерфейс Home Assistant.
- Ввод OAuth-токена через config flow.
- Автоматическое получение и обновление IAM-токена.
- Сенсор баланса Yandex Cloud.
- Отображение значения с двумя знаками после запятой.
- Единица измерения: `RUB`.
- Иконка сенсора: `mdi:cash`.
- Класс состояния: `measurement`.
- Кнопка ручного обновления баланса.
- Настраиваемый интервал обновления.
- Поддержка нескольких биллинговых аккаунтов.

## Установка

Скопируйте папку интеграции в каталог `custom_components` вашего Home Assistant.

Должна получиться такая структура:

```text
/config/custom_components/yandex_cloud_balance/
```

После копирования перезапустите Home Assistant.

## Добавление интеграции

1. Откройте Home Assistant.
2. Перейдите в **Настройки → Устройства и службы**.
3. Нажмите **Добавить интеграцию**.
4. Найдите **Yandex Cloud Balance**.
5. Введите OAuth-токен Яндекса.
6. Укажите интервал обновления в минутах.
7. Завершите настройку.

## Получение OAuth-токена

Для работы интеграции нужен OAuth-токен Яндекса, который позволяет получить IAM-токен и прочитать данные биллинга Yandex Cloud.

Интеграция использует следующие API:

```text
https://iam.api.cloud.yandex.net/iam/v1/tokens
https://billing.api.cloud.yandex.net/billing/v1/billingAccounts
```

Если OAuth-токен неверный или у него недостаточно прав, интеграция не добавится и покажет ошибку авторизации.

## Создаваемые сущности

После добавления интеграции создаётся устройство:

```text
Yandex Cloud Billing
```

Внутри устройства появляются сущности:

```text
sensor.<account_name>
button.<account_name>_refresh_balance
```

Сенсор показывает баланс в формате:

```text
293.62 RUB
```

## Атрибуты сенсора

Сенсор баланса содержит дополнительные атрибуты:

```yaml
account_id: идентификатор биллингового аккаунта
account_name: название аккаунта
status: статус аккаунта
created_at: дата создания аккаунта
```

## Ручное обновление

Для ручного обновления используется кнопка:

```text
Обновить баланс
```

При нажатии кнопка вызывает обновление координатора данных Home Assistant и запрашивает свежий баланс через API.

## Интервал обновления

Интервал обновления задаётся при добавлении интеграции.

По умолчанию:

```text
60 минут
```

Минимальное значение:

```text
5 минут
```

Максимальное значение:

```text
1440 минут
```

Интервал можно изменить в настройках интеграции.

## Пример отображения

```text
zna123    293.62 RUB
```

## Поведение при ошибках

Если API временно недоступен, Home Assistant переведёт сенсор в состояние:

```text
unavailable
```

Если OAuth-токен неверный или больше не работает, интеграция потребует повторной настройки авторизации.

## Совместимость

Интеграция написана для Home Assistant с поддержкой config flow, sensor, button и DataUpdateCoordinator.

Проверялась на:

```text
Home Assistant / HassWP 2026.3.3
```

## Файлы интеграции

Основные файлы:

- `api.py` — работа с Yandex Cloud API.
- `coordinator.py` — обновление данных через DataUpdateCoordinator.
- `sensor.py` — сенсор баланса.
- `button.py` — кнопка ручного обновления.
- `config_flow.py` — настройка через UI Home Assistant.
- `manifest.json` — описание интеграции.
